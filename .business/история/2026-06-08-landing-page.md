# 2026-06-08 — Лендинг + навигационный флоу

## 1. Задача
Создать маркетинговый лендинг как точку входа в приложение. Определить путь клиента: лендинг → регистрация / покупка / вход. Экран входа показывать поверх приложения как overlay.

## 2. Как решал
- Предложил 5 вариантов Hero-заголовков на основе болей аудитории из `.business/audience/`.
- Пользователь выбрал «Почему ваша косметика не работает?» и «Ваш личный эксперт по косметике».
- Создал `src/Landing.jsx` (7 секций: Navbar, Hero, Проблемы, Решение, Шаги, Отзывы, Тарифы, CTA, Footer).
- Обновил `src/DesignPreview.jsx`: добавил state-machine (screen + modal), модальные оверлеи (LoginModal, RegisterModal, PurchaseModal).
- Фиксил dark mode (добавил `color-scheme: light`, явный фон на body).
- Фиксил логику перехода: при клике «Войти» — мгновенный переход на app-экран + модал поверх него (убрал `AnimatePresence mode="wait"` в пользу условного рендера).

## 3. Результат
Да. Всё работает и проверено в превью:
- Лендинг с Playfair Display, без эмодзи, стеклянный стиль
- Все 7 секций (4 850px итоговая высота)
- «Войти» → app visible за dimmed backdrop → login modal в центре viewport
- «Попробовать бесплатно» → register modal
- «Оплатить подписку» → purchase modal

## 4. Эффективность
Решение хорошее. Можно было раньше понять про dark mode в preview-инструменте — потратил несколько итераций. Также `AnimatePresence mode="wait"` создавал неправильный UX при показе модала — нужно было убрать сразу.

## 5. До / После
- До: `DesignPreview.jsx` показывал только вкладку «Главная» как часть приложения.
- После: полноценный маркетинговый лендинг с путём клиента, модальными оверлеями и переходом в приложение.
