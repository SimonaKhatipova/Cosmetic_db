# Hero redesign + мульти-шаговая регистрация — 2026-06-08

## Задача

Расширить Hero лендинга: сделать правую колонку значительно информативнее.
Добавить аватары в анимированный каталог, уход для волос как приоритет (пронумерованные шаги),
шкалу безопасности состава, мини-сравнение средств, прогресс-бар теста, обновлённые метрики.
Убрать длинные тире из текстов. Общаться с пользователем гендер-нейтрально на «Вы».
Построить мульти-шаговый модал регистрации: email → код → онбординг.

## Как решал

**Landing.jsx:**
- Добавил поля `init` (инициалы) в CATALOG_A/B, отображаю в MiniCard как цветной квадрат с буквами
- Создал `ProcessingBar` — анимированная шкала с 4 фазами, переключается каждые 2.2с через useEffect
- Создал `ComparisonMini` — мини-таблица 4 продукта × 3 строки (безопасность, ингредиенты, совпадение)
- Создал `SafetyBadge` — зелёный круг 100, подпись «Высокая», тег «Супер состав»
- Реорганизовал правую колонку: каталог → волосы (приоритет) → safety|comparison → лицо → прогресс
- Уход для волос: профиль-тег + 4 нумерованных шага (Очищение, Питание, Восстановление, Финиш)
- Сетку правой колонки расширил до `minmax(360px, 510px)`, hero до `maxWidth: 1260`
- Метрики слева: 20 000+, 1 200+, 97%
- Все em-тире (—) убраны, предложения реструктурированы. «Вы/Вас/Ваш» везде заглавное

**DesignPreview.jsx:**
- RegisterModal полностью переписан на 3 шага через `useState(step)`
- Шаг 1: email + список того, что даётся бесплатно + кнопка «Получить код»
- Шаг 2: «Проверьте почту» + крупный input для 6-значного кода + back-кнопка
- Шаг 3: онбординг — BH-иконка, список с зелёными чекмарками (доступно) и серыми ⊘ (подписка), две кнопки (начать / оформить)
- Прогресс-бар из 3 сегментов в шапке каждого шага

## Результат

Да, реализовано полностью.

## Оценка

Эффективно. Hero стал значительно богаче визуально без перегрузки.
Уход для волос как приоритетный продукт выделен наглядно.
Мульти-шаговая регистрация убедительная — код подтверждения добавляет доверие.
Онбординг честно разграничивает free/paid, уместно предлагает апгрейд.

## Как было / как стало

Было: одна компактная правая колонка со скроллящим каталогом и простыми блоками ухода.
Стало: насыщенная колонка с 5 блоками разного типа (каталог, шаги, safety, сравнение, прогресс).
Регистрация: 1 экран → 3-шаговый флоу с кодом и онбордингом.
