# Правки v6 — лаконичное меню, мобильный UX, иконка на экран, фильтр-иерархия

**Дата:** 2026-06-14
**Статус:** в работе
**Контекст:** доработки после прошлой сессии. Beauty Helper — отдельный продукт; О Симоне / Интенсив / Консультация / Подборки — другие продукты со своими лендингами (на одном домене позже), их надо убрать с лендинга BH. Плюс: серьёзный мобильный UX, иконка «на экран Домой», иерархия фильтра «вид средства».

## Фазы

- [x] **Фаза 1 — лаконичное стеклянное меню BH.** Навбар переписан: Бренд · Тарифы · Войти · Попробовать (CSS-классы `.lp-nav`/`.lp-brand`/`.lp-nav-btn`, стекло blur 22px). Удалены страницы `AboutPage`/`IntensivePage`/`ConsultPage`, их хелперы (~290 строк) и весь хеш-роутинг; футер очищен от навигации по доп.лендингам. Доп. продукты (О Симоне/Интенсив/Консультация/Подборки) — отдельные лендинги в будущем, с реальными текстами. Проверено в preview.
- [x] **Фаза 2 — метрики на лендинге.** Было 2 цифры, стало 4: «20 000+ ингредиентов», «1 200+ средств», «20+ типов средств», «100% составов на русском». Десктоп — в ряд с разделителями, мобайл — 2×2 (`.lp-stats`/`.lp-stat`).
- [x] **Фаза 3 — мобильный UX лендинга.** Вертикальные отступы секций переведены на `clamp()` (96→clamp(56–96), 80→clamp(52–80), hero-top → clamp(88–112)). Бренд в навбаре `clamp(17–20px)`, кнопки компактнее на ≤600/≤360. Hero-грид сворачивается в колонку, горизонтального overflow нет на 320/375/414. Проверено.
- [x] **Фаза 4 — мобильная навигация приложения.** Добавлена нижняя навигация `.bottom-nav` (Средства/Ингредиенты/Аналоги/Сравнение, SVG-иконки, бейдж количества на «Сравнение»). На ≤600px верхние табы скрыты, контенту добавлен `padding-bottom`, бренд в шапке мельче. Проверено в preview (мок-данные).
- [x] **Фаза 5 — «Добавить на экран Домой» (iOS).** Брендовая иконка `public/app-icon.svg` → PNG 192/512 + `apple-touch-icon.png` (через qlmanage+sips). `index.html`: apple-touch-icon, manifest, theme-color, apple-mobile-web-app-*, lang=ru, viewport-fit=cover. `public/manifest.json` (standalone, иконки). Компонент `AddToHomeScreen` — bottom-sheet только для Safari на iOS (после согласия с cookie), с выгодами и шагами; закрытие запоминается; флаг `bh_a2hs_force` для предпросмотра. Проверено через force.
- [x] **Фаза 6 — фильтр «вид средства» (десктоп).** Дерево раскрыто по умолчанию (как и было). Переписан CSS `.hier-tree-*`: группы — жирные заголовки с иконкой и разделителем сверху; подгруппы — с акцентной линией слева; типы — отступ и меньший вес. Три чётких уровня. Проверено в preview.

## Гейты

`npm run build` ✓. `npm run lint` — без новых ошибок в новом коде. Проверено в preview: лендинг (десктоп/375), метрики 2×2, приложение (нижняя навигация, иерархия фильтра на 1280), модалка A2HS, отсутствие горизонтального overflow на 375. Авторизованная часть и каталог проверялись через временный обход входа + мок-данные (удалены до коммита).

## Итог

**Реализовано целиком (фазы 1–6).** Beauty Helper-лендинг очищен до своего продукта (лаконичное стеклянное меню, доп.лендинги вынесены на будущее), добавлены метрики, серьёзно доработан мобильный UX (флюидные отступы, нижняя навигация вместо десктопных вкладок, масштаб логотипа), сделана иконка и модалка «на экран Домой» для iOS, наведена иерархия в фильтре «вид средства».
