# INCI-матчер v2: нормализация + алиасы при добавлении продукта

**Дата:** 2026-06-12
**Статус:** реализован (не закоммичен, ручная проверка за Симоной)
**Контекст:** после чистки данных v4 (скрипты `migration_v4/008–013`) согласовано: новые продукты не грузим, пока матчер в `App.jsx` не перестанет плодить дубли. Сейчас при сохранении продукта ингредиент ищется по точному `inci_name` (только без учёта регистра) — алиасы и нормализация игнорируются.

## Проблема

`AddProductModal.save()` (`src/App.jsx`, ~3303):
```js
const found = ingredients.find(ing => ing.inci_name?.toLowerCase() === inciName.toLowerCase());
```
- не смотрит в колонку `aliases` (канонический ингредиент один, варианты — через `|`);
- не нормализует написание (дефисы, скобки, концентрации, кириллические гомоглифы, OCR-разрывы);
- не заполняет `match_source`.

Каждый импорт снова создаёт unmatched-строки и провоцирует дубли — ровно то, что чистили скриптами 008–013.

## Решение

Перенести в JS проверенную формулу нормализации из SQL-чистки (точное соответствие `011_dedupe_final.sql` и `013_ocr_spaces.sql`):

1. **Строгий ключ** (`inciKey`): lower → транслит гомоглифов `авекмнорстух`→`abekmhopctyx` → `_-*`→пробел → убрать `(...)` → срезать хвост-концентрацию `\d+(ppm|ppb|%|iu/g)$` → схлопнуть слэши и пробелы.
2. **Агрессивный ключ** (`inciKeyAggressive`, для OCR): транслит → убрать `_-*`, не-ASCII, скобки → оставить только `[a-z0-9/]`; матчить только при `length >= 4` и единственном кандидате.
3. **Индекс** (`buildInciIndex`): Map по ключам от `inci_name` + каждого алиаса. Коллизии в агрессивной карте → запись помечается неоднозначной и не матчится.
4. **Матчинг** (`matchInci`): exact (lower) → строгий ключ → агрессивный ключ. Возвращает ингредиент + `match_source` (`exact` / `norm` / `ocr`).
5. В `save()` писать `match_source`; `raw_inci_name` сохраняется как есть (концентрации значимы — не терять).
6. UI: в списке распознанных ингредиентов показать статус матча (найден в справочнике / не найден), чтобы опечатки правились до сохранения.

## Не входит (non-goals)

- Поиск по алиасам в каталоге ингредиентов (вкладка «Ингредиенты») — отдельная мелкая доработка.
- Перепривязка существующих ~83 unmatched-строк (это мусор парсинга, решено оставить).
- Редактирование состава существующего продукта (в UI его нет).

## Итог

**Реализован целиком.** Матчер при добавлении продукта теперь: exact → нормализованный ключ (формула 011) → агрессивный OCR-ключ (формула 013, только при единственном кандидате), смотрит в `aliases`, пишет `match_source` (`exact`/`norm`/`ocr`), сохраняет `raw_inci_name` нетронутым (концентрации не теряются). UI показывает, что нашлось в справочнике, до сохранения.

Осталось вне плана:
- ручная проверка в живом приложении (добавить тестовый продукт, проверить `match_source` в БД) — за Симоной;
- блокировка импорта снята: после ручной проверки можно грузить новые продукты;
- поиск по `aliases` во вкладке «Ингредиенты» («Fragrance» в поиске каталога по-прежнему не найдёт Parfum) — отдельная мелкая задача.
