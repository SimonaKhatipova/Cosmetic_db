# Объединение лендинга с боевым приложением

**Дата:** 2026-06-12
**Ветка:** test/full-app
**Статус:** реализован (вход по паролю и OTP-письмо — ручная проверка за Симоной)

## Проблема

На ветке test/full-app `npm run dev` показывает дизайн-прототип `DesignPreview`, а не боевое приложение:
- `src/main.jsx` рендерит `DesignPreview` вместо `App`;
- модалка входа в прототипе — фейковая (`setTimeout`, без Supabase), не та, что в боевом `App.jsx` (полноэкранный `LoginScreen` с маскотом);
- после «входа» показывается заглушка `AppPlaceholder` («это превью рабочего приложения») вместо реального каталога.

При этом у прототипа есть функции, которых нет в боевом входе: регистрация (3 шага: email → код → онбординг), переключение вход↔регистрация, модал покупки подписки (CloudPayments).

## Решение

Точка входа — боевой `App`. Для неавторизованных — лендинг, с него:
- «Войти» → боевой `LoginScreen` (маскот, реальный Supabase signIn) + новые ссылки «Зарегистрироваться» и «← На главную»;
- «Попробовать бесплатно» → новый `RegisterScreen` в боевом стиле (login3 + маскот): 3 шага с прогресс-баром, **реальная** регистрация через Supabase email-OTP (`/auth/v1/otp` → `/auth/v1/verify`), онбординг со списком free/pro;
- «Оформить подписку» → `PurchaseModal` в боевом стиле модалок, оплата через `openPaymentWidget` (CloudPayments);
- после входа/регистрации — реальное приложение (как сейчас при `authed`);
- «Выйти» → возврат на лендинг.

`DesignPreview` остаётся дизайн-песочницей (`npm run design`), из боевого флоу исключается.

## Ограничения и заметки

- Письмо с 6-значным кодом: шаблон «Magic Link» в Supabase Dashboard → Auth → Email Templates должен содержать `{{ .Token }}` (по умолчанию там только ссылка). Без этого пользователь получит письмо без кода.
- Запись факта подписки в БД (таблицы подписок нет) и freemium-гейтинг — НЕ в этом плане (фаза 3 плана лендинга `2026-06-08-landing-page.md`).
- CloudPayments работает только при заданном `VITE_CLOUDPAYMENTS_PUBLIC_ID` в `.env`; без него модалка покажет понятную ошибку.

## Итог

**Реализован целиком.** `npm run dev` теперь показывает боевой флоу: лендинг → вход (правильная модалка с маскотом) / регистрация (3 шага, реальный Supabase OTP) / покупка (CloudPayments) → реальное приложение. `DesignPreview` остался дизайн-песочницей (`npm run design`).

Ручная проверка за Симоной:
1. вход с реальными креденшелами → каталог;
2. регистрация: прежде настроить шаблон письма в Supabase Dashboard → Auth → Email Templates → «Magic Link»: добавить `{{ .Token }}` (иначе письмо придёт без 6-значного кода);
3. для оплаты — `VITE_CLOUDPAYMENTS_PUBLIC_ID` в `.env`.

Не входит (следующие шаги): freemium-гейтинг (фаза 3 плана лендинга), запись факта подписки в БД (нужна таблица подписок + проверка на бэкенде).
