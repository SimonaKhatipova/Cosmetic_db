# plans/ — технические планы функций

Здесь хранятся планы реализации каждой новой функции проекта.

## Правила

- **Один план = одна функция.** Не смешивать несколько задач в одном файле.
- **Имя файла:** `YYYY-MM-DD-название-функции.md` (дата начала работы).
- **Структура плана:** делится на фазы; у каждой фазы статус `[ ]` (не сделано) или `[x]` (готово).
- **Итоговый блок в конце:** реализован ли план целиком, что осталось, какие решения приняты.
- **Агент обязан актуализировать план** после каждой сессии: отмечать выполненные фазы, дописывать что было сделано и почему.

## Шаблон

```
# Название функции

## Фаза 1 — [название]
- [ ] Задача

## Фаза 2 — [название]
- [ ] Задача

---
## Итог
Статус: реализован / частично / не реализован
Что осталось: ...
```
