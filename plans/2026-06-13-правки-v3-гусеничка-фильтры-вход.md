**Дата:** 2026-06-13
**Статус:** реализован
**Контекст:** третий раунд правок по фидбеку Симоны после лендинга v2 и правового оформления.

## Фазы

- [x] **Шапка лендинга:** убраны ссылки «Оферта» и «Политика» (остались Тарифы / Войти / Попробовать бесплатно). Документы по-прежнему в подвале.
- [x] **Cookie-плашка:** убран чекбокс, осталась только кнопка «Принять». Согласие пишется в `localStorage` (`bh_cookie_consent`, ISO-дата); после входа переносится в профиль пользователя (`user_metadata.consent_cookie`) функцией `syncCookieConsent()`.
- [x] **Экран входа по образцу регистрации:** общий маскот `MascotFigure` с двигающимися глазами, единый стиль карточки. Добавлена ссылка «Забыли пароль?» (`sendRecovery` → POST `/auth/v1/recover`, зелёный инфо-баннер `login3-info`).
- [x] **Клик вне формы входа/регистрации** возвращает на тот же блок лендинга: вход и регистрация рендерятся оверлеями (`reg-overlay`) поверх всегда смонтированного `<Landing>`; клик по фону закрывает оверлей без перемонтирования лендинга (позиция прокрутки сохраняется). Проверено в preview.
- [x] **Гусеничка вместо змейки** (`MetamorphosisBar`): кокон справа + 10 яблок → гусеничка вылупляется (тело в 1 сегмент) → ползёт справа налево, ест яблоки и растёт сегмент за сегментом (инчворм-волна) → на левом конце превращается в бабочку → бабочка перелетает слева направо и цикл повторяется. Подписи недель сохранены. Чистый rAF, setState только на смену недели.
- [x] **Тексты лендинга без длинного тире и клише «не это, а это»:** вычищены отзывы и прочие блоки (тире осталось только в комментариях кода). Hero-подзаголовок переписан под формулировку Симоны (вопросы про шампунь/маску/активы + «Давайте разбираться вместе с Beauty Helper»). Блок «Почему так происходит» переписан в формате боль→решение.
- [x] **Подсветка названия сервиса:** компонент `Brand()` (фирменный шрифт Familjen Grotesk + акцентный цвет) подставлен везде, где «Beauty Helper» встречается в тексте. Логотипы в шапке и подвале уже используют фирменный шрифт — оставлены как есть.
- [x] **Десктоп: фильтры слева развёрнутым списком** (`HierTypeExpanded` + сайдбар `.filter-sidebar`): вся иерархия «Вид средства» раскрыта сразу, как в каталогах; применение по клику без кнопки «Применить»; контекстные фильтры (тип кожи головы, функция, промывка) и чипы реагируют мгновенно. На мобильном (≤860px) сайдбар скрыт, фильтры остаются дропдауном по кнопке «Фильтры». Контролы фильтров вынесены в общую переменную `filterControls` (без дублирования между панелью и сайдбаром).

## Открытые вопросы Симоны (ответы)

1. **Как запоминаем приём cookie / есть ли таблица пользователей.** Отдельной таблицы профилей в БД нет — данные пользователя хранятся в Supabase Auth (`auth.users.user_metadata`: имя, телефон, факты согласий с датой, `consent_cookie`). До входа cookie-согласие лежит в `localStorage` устройства, после входа переносится в профиль. На будущее (план правового оформления): для юридически удобной выгрузки согласий завести отдельную таблицу-журнал.
2. **Сервис рассылки для подтверждения почты.** Сейчас письма (OTP-код, сброс пароля) шлёт встроенный SMTP Supabase — он жёстко лимитирован (~3–4 письма/час) и годится только для теста. Для боевого запуска нужно подключить свой SMTP-провайдер (Resend / SendGrid / Unisender) в Supabase Dashboard → Auth → SMTP. Также в шаблоне «Magic Link» должен быть `{{ .Token }}`, чтобы отправлялся 6-значный код.
