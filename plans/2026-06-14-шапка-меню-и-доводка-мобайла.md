# Правки v7 — меню-шапка приложения, возврат меню лендинга, доводка мобайла

**Дата:** 2026-06-14
**Статус:** реализовано, ждёт локального ревью Симоны (на Netlify-ветке `main`)
**Контекст:** переезд на Netlify (ветка `main`, домен simonabeauty.ru, base `/`, под Pages больше не собираем). Пакет правок после ревью.

## Фазы

- [x] **Фаза 1 — вернуть меню лендинга + доп.страницы.** Восстановлены пункты меню «О Симоне / Интенсив / Консультация» (+ Тарифы) и страницы `AboutPage`/`IntensivePage`/`ConsultPage` с хеш-роутингом (`#/about` и т.д.) — из бэкапа. Меню лаконичное, стеклянное, в шапке только бренд «Beauty Helper»; на десктопе пункты в ряд, на мобайле — бургер-дровер. Футер снова с навигацией. Заголовки продуктов в шапку не выносим — только пункты меню.
- [x] **Фаза 2 — выровнять метрики.** Блок цифр (20 000+ / 1 200+ / 20+ / 100%) переведён с flex+бордеры (плыли при изменении ширины) на ровную CSS-сетку: 4 колонки на десктопе, 2×2 на мобайле. Больше не «плавают».
- [x] **Фаза 3 — карточки каталога на мобайле.** Были по одной гигантской на строку (то самое «блоки открываются слишком большими»). Сделал 2 карточки в ряд (`.grid` → `repeat(2, minmax(0,1fr))`), компактнее тело/заголовок. Модалка средства на мобайле — нормальный скролл-лист, не трогал.
- [x] **Фаза 4 — меню-шапка приложения.** Все кнопки из шапки убраны в аккуратное стеклянное меню справа (`HeaderMenu`): триггер = аватар + тариф + шеврон. Пункты словами: Личный кабинет, Настройки, Режим редактора (только владелец), Услуги, Миссия, Служба поддержки, Выйти. Поповер рендерится `position:fixed` от координат триггера — иначе его режет `overflow:hidden` шапки. «Услуги»/«Миссия»/«Поддержка» — `InfoModal` (описание + заявка/почта mailto). Контекстные инструменты редактора (+ Средство, 🔑) остаются в шапке, но только при включённом режиме редактора. ⚙-иконки заменены словом «Настройки» (в меню и в топбаре кабинета).

## Гейты

`npm run build` ✓, `npm run lint` без новых ошибок. Проверено в preview (мобайл 375 + узкий десктоп): меню-шапка и поповер, InfoModal «Услуги», бургер лендинга со всеми пунктами, переход на «О Симоне», метрики 2×2, каталог 2 карточки в ряд. Авторизованная часть/каталог — через временный обход входа + мок (удалены до сборки).

## Деплой

По новым правилам: ветка `main` → Netlify сам собирает и публикует на simonabeauty.ru. Эту сессию Симона смотрит локально (`npm run dev`); под GitHub Pages больше не собираем. Пуш в `main` (→ деплой) — после её ревью.
