## Фазы

- [x] 1. **Полноэкранное отображение**: `index.css` был дефолтным шаблоном Vite (`#root { width: 1126px }` + рамки) — заменён на минимальный reset. Сайт тянется на любой вьюпорт.
- [x] 3. **Фон приложения**: мелкие падающие цветочки (`PetalsBackground` с 22 SVG) заменены на 5 крупных PNG-цветов с лендинга (`public/flowers/flower.png`, hue-rotate, прозрачность 0.05–0.07) + 2 размытых фиолетовых пятна. На лендинг добавлены такие же фиолетовые акценты. Попутно починен путь к цветку: был абсолютный `/flowers/flower.png`, который ломался под base `/Cosmetic_db/` (на GitHub Pages цветы не отображались вовсе) — теперь `import.meta.env.BASE_URL`.
- [x] 5. **Регистрация поверх лендинга**: `RegisterScreen` рендерится оверлеем с blur(16px) над лендингом (клик мимо карточки — возврат). В шаге 1 спрашивается имя, после подтверждения кода сохраняется в `user_metadata` (PUT /auth/v1/user). В шапке приложения — «Привет, Имя».
- [x] 6. **Кислоты — фиолетовые** (#8a63b8 вместо красного #b0524a, конфликтовавшего с отдушками).
- [x] 7. **Иерархия «Вид средства»**: трёхуровневое дерево из `product_type_tree.csv` зашито константой `PRODUCT_TYPE_TREE`. Меню: группа (Уход за волосами / Уход за кожей) → подгруппы с типами; кликом выбирается тип или целая подгруппа. Тёзки между группами (Крем, Пилинг, Спрей, Лосьон, Маска) различаются полем `source_sheet` продукта (проверено по живым данным). Типы, которых нет в дереве, попадают в «Прочее».
- [x] 8. **Поиск с опечатками**: расстояние Левенштейна с отсечкой (`levDist`) + `fuzzyIncludes` (допуск 1–3 опечатки по длине запроса, совпадение по началу слова). Двухпроходно: сначала точные совпадения, при нуле результатов и запросе ≥4 символов — ближайшие. Включено в поиск продуктов (имя+бренд) и ингредиентов (INCI+рус+алиасы). 15 unit-проверок пройдены.
- [x] 2/частично. **Безопасность**: см. `2026-06-12-аудит-безопасности.md` + `security_rls_lockdown.sql`. UI редактора скрыт для не-владельца.
